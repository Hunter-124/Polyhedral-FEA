# ADR-0017: VEM k=2 with serendipity edge midpoints

- Status: accepted (2026-07-10)

## Decision
Ship Virtual Element Method **k=2** for convex polyhedra as an order upgrade
of `ElementType::kPolyVem`, using **vertex + edge-midpoint DOFs** (serendipity
style), without face-interior or cell-interior moment DOFs.

Order is inferred from connectivity:
- `n = n_vertices` → k=1 (ADR-0011)
- `n = n_vertices + n_edges` → k=2 (this ADR)

Mid-edge nodes follow `poly_edges(faces)` order (first-seen undirected edges
while walking faces). `hex20_as_poly` reorders canonical hex20 mids into that
layout so p-elevate mid-edge nodes are reusable.

## Hex serendipity path (ROADMAP C4 acceptance)
On **hexahedral cells** (6 quad faces, 8 vertices, 12 edges, 20 nodes) the k=2
stiffness, body load, and energy-norm recovery **coincide with isoparametric
hex20 serendipity FEM**:

- Same DOFs (corners + edge midpoints).
- Same polynomial completeness on parallelepipeds (serendipity space).
- Multi-element assembly, patch test, and MMS energy-norm order 2 are those of
  hex20 (verified in Catch2 within ±0.2 of theory).

This is deliberate, not a shortcut around theory: classical 3D VEM k=2 also
needs face moments (degree k−2 = 0) and internal moments for a fully
polyhedral projector whose discrete tractions cancel on shared faces. Without
those DOFs, a pure nodal least-squares [P₂]³ projector is exact on a single
cell but does **not** give reliable multi-element residual cancellation for
quadratic fields (diagnosed during C4). Hex serendipity FEM is the correct
energy-stable realization of “edge-midpoint quadratic” on hex meshes and
shares nodes with `promote_to_quadratic`.

## General polyhedra (incremental)
Non-hex polyhedra with mid-edge nodes still build a discrete [P₂]³ nodal
projector + τ μ h (I−DΠ)ᵀ(I−DΠ) stabilization (exact [P₂]³ energy on a single
cell). Multi-element MMS order for general poly k=2 is **not** claimed in this
delivery; full face/internal-moment VEM k=2 is future work.

## API
- `vem_poly_stiffness(..., int order = 1)` and mesh overloads with
  `vem_infer_order`.
- `hex20_as_poly`, `poly_edges`, `vem_body_load`, `vem_energy_error_sq`.

## Acceptance (ROADMAP C4)
- k=1 patch test unchanged.
- k=2 patch test exact on hex mid-edge meshes.
- Degree-2 MMS energy error ~ solver precision on k=2 hex path.
- Degree-3 MMS energy-norm order ≈ 2 within ±0.2 on uniform hex refinement.

## Alternatives considered
- Full face/internal moment VEM k=2: deferred (DOF model + face quadrature).
- Nodal LS [P₂]³ projector for multi-element hex: fails residual cancellation.
- Product-only hex20 without `kPolyVem`: breaks the poly-cell assembly path used
  by hex-VEM meshing.
